# iPhone 扫码与二维码/条码生成工具

这是一个纯前端小工具，支持：

- 使用手机相机扫描二维码（QR Code）和常见条码。
- 把任意文字、URL 等内容生成二维码。
- 生成常见条码（CODE128 / EAN13 / UPC）。

## 使用方式

1. 在项目目录启动静态服务器：

```bash
python3 -m http.server 8000
```

2. 用 iPhone Safari 打开：

- 本机访问：`http://localhost:8000`
- 或同局域网访问：`http://你的电脑IP:8000`

> 提示：相机权限在 HTTPS 或 localhost 环境下更稳定。

## 如何测试该 App

下面给出一套可直接执行的测试流程（建议按顺序）：

### 1) 启动与连通性（基础冒烟测试）

```bash
python3 -m http.server 8000
curl -I http://127.0.0.1:8000/index.html
```

期望结果：返回 `HTTP/1.0 200 OK` 或 `HTTP/1.1 200 OK`。

### 2) iPhone 真机扫码测试

1. iPhone 打开 `http://你的电脑IP:8000`。
2. 点击「扫码」页签，再点「开始扫码」。
3. 首次使用允许 Safari 相机权限。
4. 用另一台设备展示一个二维码（例如任意网址），让 iPhone 摄像头对准。
5. 观察页面中的「扫码结果」是否实时显示内容。
6. 点击「停止扫码」，确认相机流停止、页面提示“已停止扫码”。

通过标准：
- 能成功拉起后置摄像头。
- 能识别至少 1 个二维码。
- 停止扫码后不再继续识别。

### 3) 二维码生成功能测试

1. 切到「生成二维码」。
2. 输入文本，例如：`Hello iPhone QR` 或 `https://example.com`。
3. 点击「生成二维码」。
4. 用 iPhone 相机或微信扫一扫扫描新生成的二维码，验证内容一致。

通过标准：
- 输入非空文本时能生成二维码图。
- 扫描结果与输入文本完全一致。

### 4) 条码生成功能测试

1. 切到「生成条码」。
2. 选择 `CODE128`，输入 `ABC123456`，点击「生成条码」。
3. 再分别测试：
   - `EAN13` + `1234567890128`（13位数字）
   - `UPC` + `123456789012`（12位数字）
4. 若输入不合法（例如 EAN13 输入字母），应看到错误提示。

通过标准：
- 合法输入能渲染条码。
- 非法输入出现可读错误提示。

### 5) 回归检查（建议）

- 多次切换三个页签后，功能仍正常。
- 连续开始/停止扫码 3 次以上，不出现卡死。
- 横屏/竖屏切换后 UI 未明显错位。

## 常见问题

- **相机打不开**：请确认是 `localhost` 或 HTTPS 环境，并检查 Safari 相机权限。
- **局域网无法访问**：确认手机和电脑在同一 Wi-Fi，且防火墙未拦截 8000 端口。
- **条码生成失败**：检查所选格式和输入长度是否匹配（EAN13=13位数字，UPC=12位数字）。

## 技术栈

- [html5-qrcode](https://github.com/mebjas/html5-qrcode)：扫码
- [qrcodejs](https://github.com/davidshimjs/qrcodejs)：二维码生成
- [JsBarcode](https://github.com/lindell/JsBarcode)：条码生成
